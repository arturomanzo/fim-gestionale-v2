🔄 FIM Gestionale — Migrazione v1 → v2

Trasferimento dati da localStorage (v1) a Supabase (v2)

1 Esporta dati dal Gestionale v1

Apri il file FIM_Insurance_Database.html nel browser, poi premi F12 → Console e incolla lo script qui sotto. Copia il risultato JSON nel campo sottostante.

Script da copiare in console (clicca per copiare):

(function(){
  var keys=['fim_clienti','fim_polizze_db','fim_collaboratori','fim_compagnie',
    'fim_sinistri','fim_fatture','fim_preventivi','fim_quotazioni',
    'fim_quietanze','fim_settings','fim_prov_table','fim_indiv_prov',
    'fim_prov_overrides','fim_polizze_saldate','fim_wa_templates'];
  var data={_exportDate:new Date().toISOString(),_version:'v1'};
  keys.forEach(function(k){
    try{var v=localStorage.getItem(k);if(v){data[k]=JSON.parse(v)}}
    catch(e){try{data[k]=v}catch(e2){}}
  });
  var j=JSON.stringify(data);
  console.log('FIM Export: '+Object.keys(data).length+' chiavi, '+j.length+' chars');
  // Copia negli appunti
  if(navigator.clipboard){navigator.clipboard.writeText(j).then(function(){
    console.log('✅ Copiato negli appunti! Incolla nel tool di migrazione.')
  })}
  // Anche download come file
  var b=new Blob([j],{type:'application/json'});var u=URL.createObjectURL(b);
  var a=document.createElement('a');a.href=u;a.download='fim_v1_export_'+
    new Date().toISOString().slice(0,10)+'.json';a.click();URL.revokeObjectURL(u);
  console.log('📥 File scaricato: '+a.download);
  return 'Export completato: '+Object.keys(data).length+' chiavi';
})()

2 Configurazione Supabase

⚠️ Usa la Service Role Key (NON la anon key) per bypassare le RLS e inserire i dati. La trovi in Supabase → Settings → API.

3 Anteprima Trasformazione

Mappatura automatica dei campi v1 → v2 con preview dei dati trasformati.

4 Importa in Supabase

Ordine: collaboratori → clienti → polizze → sinistri. Ogni record mantiene il mapping v1_id.
